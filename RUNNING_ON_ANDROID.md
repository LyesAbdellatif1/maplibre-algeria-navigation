# Running the App on Android Emulator

This guide explains how to build and run the **React Native example app** on an Android emulator on Windows.

> The Expo example app currently has a build issue in this monorepo setup (see note at the bottom). Use the React Native app instead.

---

## Prerequisites

### 1. Node.js
Install the version specified in [`.nvmrc`](.nvmrc). Using [nvm-windows](https://github.com/coreybutler/nvm-windows):
```powershell
nvm install
nvm use
```

### 2. Yarn
This project requires Yarn 4. Do NOT install via npm.
```powershell
corepack enable
corepack prepare yarn@stable --activate
```

### 3. Android Studio
Download and install [Android Studio](https://developer.android.com/studio).

During installation, make sure these are checked:
- Android SDK
- Android SDK Platform
- Android Virtual Device (AVD)

### 4. Environment Variables
Set `ANDROID_HOME` to your SDK path. Add to your PowerShell profile or set manually:
```powershell
$env:ANDROID_HOME = "$env:LOCALAPPDATA\Android\Sdk"
$env:Path += ";$env:ANDROID_HOME\platform-tools"
```

To make it permanent, set via System Properties → Environment Variables.

### 5. Windows Long Path Support (required)
Run PowerShell **as Administrator** and execute:
```powershell
New-ItemProperty -Path "HKLM:\SYSTEM\CurrentControlSet\Control\FileSystem" `
  -Name "LongPathsEnabled" -Value 1 -PropertyType DWORD -Force
```
Also enable for Git:
```powershell
git config --global core.longpaths true
```
**Restart your PC** after setting this.

### 6. CMake 4.x via Android Studio SDK Manager (required)
`react-native-reanimated` builds C++ code using CMake. The default CMake version bundled with the Android NDK (3.22.1) cannot handle the long paths generated by this monorepo on Windows. You must install CMake 4.x through Android Studio:

1. Open **Android Studio**
2. Go to **Tools → SDK Manager → SDK Tools** tab
3. Check **"Show Package Details"** (bottom-right checkbox)
4. Expand **CMake** and select the latest **4.x.x** version
5. Click **Apply → OK** and wait for the download

Then set the `CMAKE_VERSION` environment variable **permanently** so every build uses it:

1. Open **Start → search "Environment Variables"**
2. Click **"Edit the system environment variables" → Environment Variables**
3. Under **System variables** → **New**:
   - Name: `CMAKE_VERSION`
   - Value: `4.1.2` (or whatever version was installed — check with `ls "$env:LOCALAPPDATA\Android\Sdk\cmake"`)
4. Click **OK** on all dialogs

Verify the variable is set:
```powershell
echo $env:CMAKE_VERSION
# Should print: 4.1.2
```

**Why this is needed:** `react-native-reanimated` reads `CMAKE_VERSION` from the environment (`version = System.getenv("CMAKE_VERSION") ?: "3.22.1"`). CMake 3.22.1 hits a 250-character object file path limit on Windows when the project is in a deeply nested directory. CMake 4.x resolves this by using hashed short filenames for object files when paths exceed the limit.

## First-Time Setup

### Step 1 — Install dependencies
From the project root:
```powershell
yarn install
```

### Step 2 — Create an Android Emulator
1. Open **Android Studio**
2. Go to **Device Manager** (right panel or Tools → Device Manager)
3. Click **Create Device**
4. Choose a phone model (e.g. Pixel 6)
5. Select a system image — choose **API 34** (Android 14), download if needed
6. Finish and click **Play (▶)** to start the emulator

Verify the emulator is detected:
```powershell
adb devices
# Expected output:
# List of devices attached
# emulator-5554   device
```

---

## Running the App

### Step 3 — Start the emulator
Make sure the Android emulator is running before building (see Step 2).

### Step 4 — Build and run
Before running, make sure `CMAKE_VERSION` is set (if you set it as a system variable in Prerequisites step 6, open a fresh terminal so it takes effect):
```powershell
echo $env:CMAKE_VERSION   # should print 4.1.2
yarn examples:react-native android
```

This command:
1. Starts the Metro bundler (JavaScript server)
2. Compiles all native Android code (Gradle + CMake for C++ modules)
3. Installs the APK on the running emulator
4. Launches the app automatically

**First build takes 5–15 minutes** — it compiles Java, Kotlin, and C++ (Reanimated Worklets). Subsequent builds are much faster thanks to Gradle's build cache.

## Development Workflow

### Hot reload (JS changes)
JavaScript changes are automatically reflected in the running app — no rebuild needed. The Metro bundler watches your files.

### Rebuild (native changes)
If you change Android Java/Kotlin code in `android/`, you must rebuild:
```powershell
yarn examples:react-native android
```

Or rebuild directly from **Android Studio**:
1. Open `examples/react-native-app/android/` in Android Studio
2. Press **Run (▶)** — Android Studio shows the library as `mlrn` and the app as `app`

### Metro bundler and JS console logs

`yarn examples:react-native start` does more than just start Metro — it also **rebuilds and reinstalls the app** on the emulator before connecting. This means:

- `CMAKE_VERSION` must be set in the terminal you run it from (same as a full build)
- The first run will take 2–3 minutes for the incremental Gradle + CMake build
- Subsequent runs reuse the Gradle cache and complete in ~30 seconds

Run it like this:
```powershell
# Make sure CMAKE_VERSION is set (permanent system var recommended — see Prerequisites step 6)
echo $env:CMAKE_VERSION   # must print 4.1.2

yarn examples:react-native start
```

Metro starts a dev server on port 8081 and shows an interactive menu:

```
i - run on iOS
a - run on Android
r - reload app
d - open Dev Menu
j - open DevTools
```

Press **`a`** to launch on the Android emulator (or it may open automatically).

**JS console logs** appear inline in the Metro terminal output:
```
 LOG  Running "MapLibreReactNativeExample" with {"rootTag":11}
 LOG  [your console.log output here]
 WARN [warnings in yellow]
 ERROR [errors in red]
```

> Note: React Native 0.76 warns that JS logs will be removed from Metro in v0.77. Press **`j`** to open **React Native DevTools** in Chrome or Edge — this is the recommended way to view logs going forward.

**CMake path-length warnings** (e.g. `The object file directory has 193 characters`) that appear during the build are harmless — CMake 4.x handles them by using hashed short filenames. The build will still succeed.

## Troubleshooting

### `adb devices` shows nothing
- Make sure the emulator is fully started (past the lock screen)
- Try: `adb kill-server && adb start-server`

### Build fails — "No variants exist" for native libraries
Stale build artifacts from a previous failed build. Clean and retry:
```powershell
# From the project root
Remove-Item -Recurse -Force node_modules\react-native-gesture-handler\android\build
Remove-Item -Recurse -Force node_modules\react-native-reanimated\android\build
Remove-Item -Recurse -Force node_modules\react-native-safe-area-context\android\build
Remove-Item -Recurse -Force node_modules\react-native-screens\android\build
Remove-Item -Recurse -Force examples\react-native-app\android\app\build
Remove-Item -Recurse -Force examples\react-native-app\android\build
yarn examples:react-native android
```

### Build fails — CMake path too long / `mkdir C_/...`
This happens when CMake 3.22.1 is being used instead of 4.x. Two things are required:

1. `CMAKE_VERSION` environment variable must be set to `4.1.2` (see Prerequisites step 6). If you just set it as a system variable, open a **new terminal** so it takes effect.
2. Verify the correct version is active before building:
```powershell
echo $env:CMAKE_VERSION   # must print 4.1.2, not empty
```
3. If the variable is set but the error persists, delete the stale CMake cache and rebuild:
```powershell
Remove-Item -Recurse -Force examples\react-native-app\node_modules\react-native-reanimated\android\.cxx
yarn examples:react-native android
```

### `ANDROID_HOME` not set
```powershell
$env:ANDROID_HOME = "$env:LOCALAPPDATA\Android\Sdk"
```

### Metro bundler port conflict
```powershell
yarn examples:react-native start --port 8088
```

### Metro fails — `Cannot find module '@maplibre-react-native/examples/metro.shared'`
This is caused by a broken workspace symlink. Yarn 4 creates a junction for `@maplibre-react-native/examples` pointing to `examples/shared`, but if the path was ever wrong (e.g. contained a space), the junction becomes stale.

Fix:
```powershell
# Remove the broken junction
Remove-Item -Force "examples\react-native-app\node_modules\@maplibre-react-native\examples"

# Re-create it pointing to the correct absolute path
cmd /c mklink /J "examples\react-native-app\node_modules\@maplibre-react-native\examples" "C:\ProjectsRepo\maplibre-algeria-navigation\examples\shared"

yarn examples:react-native start
```

## Note on the Expo App

Running `yarn examples:expo android` currently fails in this monorepo setup because Yarn 4's autolinking cannot resolve the local workspace packages (`@maplibre/maplibre-react-native`, `@maplibre-react-native/examples`), so the `expo-module-gradle-plugin` Gradle plugin is never registered. The React Native app is the recommended way to test the library on Android.
